Local Codex Desktop repair tool

Bring moved Codex project chats back.

A small dependency-light utility for repairing local Codex Desktop project history after moving folders out of OneDrive, iCloud Drive, Dropbox, or any renamed directory.

Start Repair View on GitHub

What Goes Wrong

Codex stores project paths in several local state layers. If a project folder moves, the visible folder path may be fixed while older thread metadata still points to the old location. The result can look like missing project chats, detached threads, or threads that disappear after opening.

SQLite thread indexthreads.cwd and threads.sandbox_policy can keep stale project roots.
Session rolloutssessions/**/*.jsonl can rehydrate old workspace metadata.
Desktop stateSaved workspace roots, project order, and sidebar state can keep the old folder.

Easy Mode on Windows

Download or clone the repo, quit Codex completely, then double-click:

Start-Repair-Windows.cmd

The wizard asks for:

  • the old project folder path
  • the new project folder path
  • whether to apply after a dry-run scan
Dry-run is the default. The tool only writes changes after you confirm and it creates backups first.

Command Line

Dry run

python scripts/codex_path_repair.py ^
  --old "C:\Users\you\OneDrive\Desktop\Codex Projects\App" ^
  --new "C:\Users\you\Desktop\Codex Projects\App"

Apply

python scripts/codex_path_repair.py ^
  --old "C:\Users\you\OneDrive\Desktop\Codex Projects\App" ^
  --new "C:\Users\you\Desktop\Codex Projects\App" ^
  --apply

Dependencies

The repair engine uses only Python's standard library: sqlite3, json, pathlib, and shutil. No pip install is required.

A fully dependency-free PowerShell-only repair is not realistic for the complete fix because Codex stores project metadata in SQLite and Windows PowerShell does not include a built-in SQLite writer.

What It Changes

  • state_5.sqlite: thread cwd and sandbox workspace metadata
  • .codex-global-state.json: saved project roots, project order, active roots, sidebar groups
  • config.toml: trusted project paths
  • sessions and archived_sessions: metadata path fields
  • process_manager/chat_processes.json: stale cwd values
